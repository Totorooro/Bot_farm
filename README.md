# Bot Farm

Микросервис-ботоферма — REST API для управления тестовыми учётными записями пользователей, используемыми в E2E-тестировании. Сервис позволяет создавать пользователей, получать их список, а также выдавать и освобождать пользователей для нужд тестовых сценариев.

## Описание

Ботоферма — вспомогательный сервис, предоставляющий реальные учётные данные пользователей для E2E-тестов. Платформа тестирования обращается к сервису за свободным пользователем, который затем авторизуется в целевом приложении и выполняет тестовые сценарии.

Ключевая концепция — **блокировка пользователя**: при выдаче пользователя на него устанавливается временная метка (`locktime`), предотвращающая его повторное использование в параллельных тестах. По истечении таймаута блокировка снимается автоматически. Освободить всех пользователей можно явно через соответствующий эндпоинт.

## Модель данных

### USER

| Поле | Тип | Описание |
|---|---|---|
| `id` | UUID | Первичный ключ |
| `created_at` | timestamp | Дата создания записи |
| `login` | string | Почтовый адрес пользователя |
| `password` | string | Пароль в зашифрованном виде (bcrypt) |
| `project_id` | UUID | UUID проекта, к которому принадлежит пользователь |
| `env` | enum | Окружение: `prod`, `preprod`, `stage` |
| `domain` | enum | Тип пользователя: `canary`, `regular` |
| `locktime` | timestamp | Временная метка блокировки; `null` — пользователь свободен |

## API

### Создать пользователя

```
POST /users
```

Создаёт нового пользователя в ботоферме. Пароль принимается открытым текстом и сохраняется в зашифрованном виде.

---

### Получить список пользователей

```
GET /users
```

Возвращает список всех пользователей. Поддерживает фильтрацию по `project_id`, `env`, `domain`.

---

### Получить (заблокировать) пользователя

```
POST /users/lock
```

Ищет первого незанятого пользователя (у которого `locktime` равен `null` или истёк), устанавливает `locktime` и возвращает его. Таймаут блокировки задаётся переменной окружения `LOCK_TIMEOUT_MINUTES`.

---

### Освободить всех пользователей

```
POST /users/free
```

Сбрасывает `locktime` у всех пользователей в системе.

## Конфигурация

Переменные окружения задаются в файле `.env` (для локального запуска) или `.env-non-dev` (для Docker Compose).

Пример конфигурации (`.env.example`):

```env
DB_HOST=db
DB_PORT=5432
DB_NAME=bot_farm
DB_USER=postgres
DB_PASS=postgres
LOCK_TIMEOUT_MINUTES=10
```

| Переменная | Описание |
|---|---|
| `DB_HOST` | Хост PostgreSQL |
| `DB_PORT` | Порт PostgreSQL |
| `DB_NAME` | Имя базы данных |
| `DB_USER` | Пользователь базы данных |
| `DB_PASS` | Пароль базы данных |
| `LOCK_TIMEOUT_MINUTES` | Время в минутах, по истечении которого блокировка пользователя снимается автоматически |

## Запуск через Docker Compose

Сервис разворачивается в двух контейнерах: `db` (PostgreSQL 15) и `bot_f` (приложение). Приложение ожидает готовности базы данных через healthcheck перед стартом.

```bash
# Скопировать и заполнить конфигурацию
cp .env.example .env-non-dev

# Собрать и запустить
docker compose up --build
```

После запуска API доступен на `http://localhost:8081`.

## Локальный запуск

Требования: Python 3.11+, запущенный экземпляр PostgreSQL.

```bash
# Установить зависимости
pip install -r requirements.txt

# Скопировать и заполнить конфигурацию
cp .env.example .env

# Применить миграции
alembic upgrade head

# Запустить сервер
uvicorn app.main:app --reload --port 8000
```
